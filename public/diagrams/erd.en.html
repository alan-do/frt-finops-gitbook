Data Model · Entity Relationship Diagram

FinOps Data Model — Full ERD

Hybrid model: PostgreSQL (OLTP) for rules / budgets / state, ClickHouse (OLAP) for billing. Every attribute is described. Use the buttons to switch view.

Source: /architecture/erd

OLTP relationships (PostgreSQL)
TEAM1 ──< NCLOUD_ACCOUNTa team owns many cloud accounts
TEAM1 ──< NBUDGETa team has many budgets (per period)
TEAM1 ──< NALLOCATION_RULEa team is the target of many rules
TEAM1 ──< NUSERa team has many member users
TEAM1 ──< NANOMALY_ALERTalerts are raised against a team
BUDGET1 ──< NANOMALY_ALERTa budget breach triggers alerts
ANOMALY_ALERT1 ── 1AGENT_ACTIONan alert is investigated by an action
USER1 ──< NAGENT_ACTIONa user approves / rejects actions
USER1 ──< NAUDIT_LOGevery change is performed by a user
OLAP relationships (ClickHouse)
CLOUD_ACCOUNT1 ──< NFOCUS_NORMALIZEDan account incurs billing records (via sub_account_id)
FOCUS_NORMALIZED1 ──< NALLOCATED_COSTeach raw record generates allocated rows
ALLOCATION_RULE1 ──< NALLOCATED_COSTa rule defines how cost is allocated
ALLOCATED_COST1 ──< NCOST_DAILY_SUMMARYallocated rows aggregate into daily rollups

## DQ rules (run pre-persist, fail-closed)

### DQ-1 ¬∑ Duplicate resource detection
For a billing period, group by stable `ResourceId`. If two rows share a normalized service signature (name with group-prefix stripped and tokens sorted), flag as a **duplicate candidate** (catches D1, D2). The pipeline halts; the row is not silently summed.

```
signature(name) = sortTokens(stripGroupPrefix(name)).join('|')
"FPT.AI-eKYC-D·ªãch v·ª•" and "D·ªãch v·ª•-FPT.AI-eKYC" ‚Üí same signature ‚Üí FLAG
```

### DQ-2 ¬∑ Empty / malformed name guard
Reject any row whose cleaned `ServiceName` is empty after trimming the group prefix and surrounding dashes (catches D3, D4). The cost is **not dropped** ‚Äî it is routed to `__UNALLOCATED__` with a reason code `EMPTY_SERVICE_NAME`, so it stays visible and zero-sum holds.

### DQ-3 ¬∑ Cross-group uniqueness
A `ResourceId` must belong to exactly one canonical domain. If the same logical service (e.g. `Public IP`) resolves to two groups, the higher-priority mapping rule wins deterministically and the conflict is logged (catches D5).

### DQ-4 ¬∑ Drop accounting (no silent loss)
Any row excluded from allocation (junk, empty group) must be **counted and logged**, not deleted (catches D6). The count feeds the **Completeness check**: `rawRowCount == allocatedRowCount + unallocatedRowCount + loggedDropCount`.

### DQ-5 ¬∑ Per-currency zero-sum
After allocation, for **each** `BillingCurrency` independently: `Œ£ allocated.BilledCost == Œ£ source.BilledCost`. FPT Cloud (VND) and LLM APIs (USD) are never summed together. A mismatch in any currency halts the write.

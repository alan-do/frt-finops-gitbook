# Service Catalog & FOCUS Mapping

<Callout type="info" emoji="🗂️">
  Phần này biến dữ liệu research dịch vụ FPT Cloud của team thành input có cấu trúc cho Core Engine: bảng mapping vendor → FOCUS, một canonical taxonomy cho báo cáo, và các quy tắc data-quality bảo vệ invariant zero-sum.
</Callout>

## Dữ liệu nguồn

Catalog lấy từ dashboard **DataSuite** (pageId 4975), dimension `dim_service`, với cấu trúc 2 tầng: **CRM Service Group × Service Mapping**.

- **33 nhóm / 145 dịch vụ** (đã loại 2 dòng rác nhóm rỗng).
- Cấu trúc 2 tầng map thẳng vào FOCUS: `Service Group → ServiceCategory`, `Service Mapping → ServiceName`.

## Vì sao quan trọng với Core Engine

Đây là input thật cho [Allocation Engine](/architecture/core-engine) tất định và phần FOCUS mapping của connector. Research cũng phát hiện các vấn đề data-quality cụ thể, validate 3 quyết định đã được groom vào spec:

| Phát hiện trong dữ liệu | Cơ chế bảo vệ trong Core Engine |
| :--- | :--- |
| Tên dịch vụ rác/rỗng (` - CPU`, `Voicebot - `), 2 dòng đã bị loại | Bucket `__UNALLOCATED__` (giữ zero-sum) |
| Tên đảo / trùng (`eKYC` cả 2 chiều, `Model as a Service` lặp) | Match theo `ResourceId` ổn định, không theo tên |
| Phân loại chồng chéo (Database ở 3 nhóm, Public IP ở 2) | Tầng canonical category trên các nhóm thô |
| FPT Cloud tính bằng VND vs LLM API bằng USD | Zero-sum check theo từng tiền tệ |

# Roadmap, Estimation & Backlog

This plan is based on the team's actual resource capacity of **1.5 FTE** (1 Full-time, 1 Part-time 50%). The core strategy is **strict adherence to the FOCUS (FinOps Open Cost & Usage Specification) standard** to normalize data from the very beginning, rather than spending effort building an internal proprietary data model.

<Tabs items={['🗺️ Roadmap Details', '📊 Estimation & 5 Core Stories']}>
  <Tab>
    Below are the detailed action items chronologically for Q3/2026, aligned closely with the FOCUS integration process.
    
    <Steps>
      ### M1: FOCUS Data Foundation (To be completed by 20/06/2026)
      **Objective:** Initialize data architecture based on the FOCUS standard.
      - FinOps Lead researches and maps required data fields according to FOCUS 1.4 (e.g., `BilledCost`, `ChargeCategory`, `ProviderName`).
      - Finalize Billing Access to AWS, Azure, and other SaaS/AI providers.
      - Establish a Database Schema that 100% complies with the FOCUS Data Model.

      ### M2: Data Ingestion & Alerts (04/07/2026)
      **Objective:** Synchronize and normalize multi-source data, combined with basic alerting.
      - **Task 1:** Deploy Data Ingestion Agents to automatically pull data from Cloud/SaaS providers and transform them into the FOCUS format.
      - **Task 2:** Cleanse data and normalize the `SubAccountId` column to accurately map to individual Product Teams.
      - **Task 3:** Set up a Webhook to send automated notifications to internal chat channels/Email when a team's `BilledCost` exceeds 110% of the baseline.

      ### M3: Unified Dashboard MVP (18/07/2026)
      **Objective:** Visualize cross-platform data leveraging FOCUS consistency.
      - **Task 1:** Deploy Grafana / Metabase connected directly to the FOCUS Data Lake.
      - **Task 2:** Build a normalized Dashboard displaying Multi-Cloud total costs on a single chart without terminology discrepancies.
      - **Task 3:** Conduct an internal Demo with the CTO & POs to demonstrate the transparent reporting capabilities enabled by FOCUS.

      ### M4: GenAI Integration (15/08/2026)
      **Objective:** Apply AI to analyze FOCUS data.
      - **Task 1:** Configure Prompts so GenAI can automatically write SQL queries against the standard FOCUS table structures.
      - **Task 2:** Request GenAI to summarize the reasons for cost spikes based on FOCUS columns like `ResourceName` and `ServiceCategory`.
      - **Task 3:** Automatically send this summary report every Monday morning via internal channels.

      ### M5: Production & ROI Tracking (30/09/2026)
      **Objective:** Final packaging and company-wide rollout.
      - Combine FOCUS data with Business Value data (User/Revenue) from teams to calculate ROI.
      - Hand over documentation (User Manual).
      - Grant Dashboard access to all Product Owners across FRT.
    </Steps>

    ### 2. List of 5 Core User Stories (M1 Backlog)

    Below are the 5 core Stories defined by the Product/Tech Lead on Jira, ready for Sprint Planning:

    <Cards>
      <Card title="🎫 Story 1: Applying the FOCUS Standard (Epic 1)" href="#">
        **As a** FinOps Data Engineer,  
        **I want to** synchronize and transform all costs from AWS and OpenAI into the standard FOCUS Format,  
        **So that** the system has a cross-platform Single Source of Truth without billing terminology conflicts.

      <Card title="🎫 Story 2: Cost Tagging (Epic 1)" href="#">
        **As a** FinOps Admin,  
        **I want to** utilize standard FOCUS data fields (such as `Tags` and `SubAccountId`) to map with the list of Product Teams,  
        **So that** the system can automatically and accurately perform Cost Allocation (Chargeback) for each department.
      </Card>
      <Card title="🎫 Story 3: Budget Overrun Alerts (Epic 2)" href="#">
        **As a** Product Owner,  
        **I want to** receive immediate alerts via internal chat channels/Email if my team's `BilledCost` column exceeds the allowed threshold,  
        **So that** I can promptly investigate the cause and halt any waste.
      </Card>
      <Card title="🎫 Story 4: Visual Monitoring (Epic 4)" href="#">
        **As a** CTO,  
        **I want to** have a comprehensive Dashboard displaying Multi-Cloud costs (running on FOCUS-normalized data),  
        **So that** I can compare costs transparently and make quick budget coordination decisions.
      </Card>
      <Card title="🎫 Story 5: AI-Driven Reporting (Epic 3)" href="#">
        **As a** Management Team member,  
        **I want to** receive automated reports from GenAI (based on AI querying directly into the FOCUS data tables),  
        **So that** I understand the technical reasons behind the numbers without needing a meeting with the Dev team.
